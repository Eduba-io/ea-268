01 · How we would read the stack

Three layers, one question about each.

Most organizations automate the wrong things. The Eduba frame is simple. Around 60% of what a Finance org does every day is traditional code and database work. Around 30% is rule-based logic. Around 10% is a real AI problem. Sorted into the right layer, the Joule rollout lands clean. Sorted wrong, it creates audit risk.

60%Database & code

Expense line items, invoice headers, GL coding, approval queues, tax jurisdiction lookups. ONTAP for Finance. Fast, boring, auditable. Not the place for an agent.

30%Rule-based logic

Policy enforcement, approval thresholds, exception routing, VAT treatment, mileage caps. Joule Studio belongs here. Rules explicit, auditable, versioned. Not a black box.

10%Genuine AI

Receipt-to-report generation, anomaly detection, pre-submit audit, conversational travel policy guidance. The Joule agent surface. This is where the change-management work concentrates.

03 · The governance research

Ethics Engine.

A psychometric assessment tool for evaluating ideological and moral patterns in LLMs. When an agent can auto-populate an expense report, auto-approve an invoice, or auto-categorize a transaction for tax treatment, the question an auditor asks is not whether the agent is accurate. The question is how the agent behaves under edge cases, under pressure, under policy ambiguity. Ethics Engine is the evaluation surface.

arxiv.org/abs/2510.11742 · github.com/RinDig/AuditEngine

Companion methodology: Interpretable Context Methodology (ICM), submitted to ACM TiiS. Folder-as-architecture for Joule Studio agent configurations.
